超越权限提示：让 Claude Code 更安全、更自主

Beyond permission prompts: making Claude Code more secure and autonomous | Anthropic | 2025-10-20

Claude Code 的新沙箱功能（沙箱化 Bash 工具和 Web 版 Claude Code）通过启用文件系统和网络隔离这两道边界，减少权限提示并提升用户安全性。

在 Claude Code 中，Claude 会与你并肩编写、测试和调试代码，浏览你的代码库、编辑多个文件，并运行命令来验证自己的工作。让 Claude 拥有如此多的代码库和文件访问权限可能带来风险，尤其是在遭遇提示词注入时。

为此，我们基于沙箱为 Claude Code 引入了两项新功能，它们既能为开发者提供更安全的工作环境，也让 Claude 以更少的权限提示更加自主地运行。在我们的内部使用中，我们发现沙箱可以安全地将权限提示次数减少 84%。通过界定 Claude 可以自由工作的边界，我们既提升了安全性，也增强了自主性。

在 Claude Code 中保障用户安全

Claude Code 采用基于权限的模型运行：默认情况下它是只读的，这意味着在进行修改或运行任何命令前都会征求你的许可。也有一些例外，例如我们会自动允许 echo 或 cat 这类安全命令，但大多数操作仍然需要明确批准。

不断点击“批准”会拖慢开发节奏，还可能导致“批准疲劳”，让用户无法仔细关注自己正在批准的内容，从而降低开发安全性。

为了解决这一问题，我们为 Claude Code 启用了沙箱功能。

沙箱：更安全、更自主的方式

沙箱会创建预设的边界，使 Claude 可以在其中更自由地工作，而不是对每个操作都请求许可。启用沙箱后，你会看到权限提示大幅减少，同时安全性提升。

我们的沙箱方案构建在操作系统层特性之上，建立了两道边界：

  1. 文件系统隔离 ,**** 它确保 Claude 只能访问或修改特定目录。这对防止遭遇提示词注入的 Claude 修改敏感系统文件尤为重要。
  2. 网络隔离 ,**** 它确保 Claude 只能连接到已批准的服务器。这可以阻止遭遇提示词注入的 Claude 泄露敏感信息或下载恶意软件。

值得注意的是，高效的沙箱需要同时具备文件系统隔离和网络隔离。没有网络隔离，被攻破的代理可能会窃取 SSH 密钥等敏感文件；没有文件系统隔离，被攻破的代理可以轻松逃离沙箱并获取网络访问权限。正是通过同时采用这两种技术，我们才能为 Claude Code 用户提供更安全、更高效的智能体体验。

Claude Code 中的两项全新沙箱功能

沙箱化 Bash 工具：无权限提示的安全 Bash 执行

我们推出了 新的沙箱运行时，目前以研究预览形式提供测试版本，允许你精确指定代理可以访问的目录和网络主机，而无需付出启动和管理容器的开销。它可以用于沙箱化任意进程、代理和 MCP 服务器，并以 开源研究预览 的形式提供。

在 Claude Code 中，我们使用该运行时为 Bash 工具提供沙箱，使 Claude 能在你设定的限制内运行命令。在安全的沙箱内，Claude 可以更自主地工作，并在无需权限提示的情况下安全执行命令。如果 Claude 试图访问 沙箱之外 的内容，你会立即收到通知，并可选择是否允许。

我们构建的方案基于操作系统级原语，例如 Linux bubblewrap 和 MacOS seatbelt，在操作系统层实施这些限制。它不仅覆盖 Claude Code 的直接操作，还覆盖命令所衍生的任何脚本、程序或子进程。如上所述，该沙箱同时强制执行：

  1. 文件系统隔离， 允许对当前工作目录进行读写，但阻止修改其之外的任何文件。
  2. 网络隔离， 仅允许通过连接到沙箱外部代理服务器的 Unix 域套接字访问互联网。该代理服务器会对进程可连接的域名实施限制，并在请求新域名时处理用户确认。如果你希望进一步提升安全性，我们也支持自定义该代理来对出站流量施加任意规则。

两大组件都可以配置：你可以轻松地允许或拒绝特定文件路径或域名。

沙箱确保即便提示词注入成功也会被完全隔离，不会影响整体的用户安全性。这样一来，被攻破的 Claude Code 既无法窃取你的 SSH 密钥，也无法与攻击者的服务器通信。

要使用这一功能，只需在 Claude Code 中运行 /sandbox，并查阅我们有关安全模型的 更多技术细节。

为了让更多团队构建更安全的代理，我们已经将这一功能 开源。我们也鼓励其他团队在其代理中采用这项技术，以增强代理的安全防护能力。

Web 版 Claude Code：在云端安全运行 Claude Code

今天我们还发布了 Web 版 Claude Code，让用户能够在云端的隔离沙箱中运行 Claude Code。Web 版 Claude Code 会在隔离沙箱中执行每个会话，在安全的前提下完全访问其服务器。我们设计的沙箱确保敏感凭证（如 git 凭证或签名密钥）绝不会与 Claude Code 同处沙箱内。这样，即使沙箱中的代码遭受攻击，用户也能免受进一步伤害。

Web 版 Claude Code 使用自定义代理服务透明地处理所有 git 交互。在沙箱内，git 客户端会使用专门构建的范围凭证对该服务进行身份验证。代理会验证此凭证以及 git 交互的内容（例如确保只推送到配置好的分支），然后再附加正确的身份验证令牌，将请求发送到 GitHub。

入门指南

我们新的沙箱化 Bash 工具以及 Web 版 Claude Code，为使用 Claude 开展工程工作的开发者带来了显著的安全性和生产力提升。

要开始使用这些工具：

  1. 在 Claude 中运行 /sandbox，并查看我们的文档，了解如何配置该沙箱。
  2. 前往 claude.com/code 体验 Web 版 Claude Code。

或者，如果你正在构建自己的代理，不妨查看我们开源的沙箱代码，并考虑将其整合进你的工作流程。我们期待看到你们的成果。

想进一步了解 Web 版 Claude Code，请阅读我们的发布博文。
