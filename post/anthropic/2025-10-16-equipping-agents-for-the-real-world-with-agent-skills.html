用 Agent Skills 让智能体适应真实世界

Equipping agents for the real world with Agent Skills | Anthropic | 2025-10-16

Claude 功能强大，但真实的工作需要流程化知识和组织语境。我们推出 Agent Skills，这是一种利用文件和文件夹打造专用智能体的新方式。

随着模型能力不断提升，我们如今可以构建能够与完整计算环境交互的通用智能体。例如 Claude Code 可以通过本地代码执行和文件系统，在多个领域完成复杂任务。但随着这些智能体变得更强，我们需要更具组合性、可扩展性和可移植性的方式，为它们注入领域专长。

因此，我们打造了 Agent Skills：这是由指令、脚本和资源组成的有序文件夹，智能体可以动态地发现并加载它们，从而在特定任务上表现更好。**** Skills 通过把你的专业知识封装成 Claude 可组合使用的资源，扩展了 Claude 的能力，将通用智能体转化为符合你需求的专用智能体。

为智能体构建一个技能，就像为新员工整理一份入职指南。与其为每个用例搭建割裂的定制智能体，现在任何人都能通过捕获和共享自己的流程知识，让智能体凭借可组合的能力实现专精。在本文中，我们解释什么是 Skills、它们如何运作，并分享打造自定义技能的最佳实践。

技能的组成

为了看看 Skills 如何发挥作用，我们来走查一个真实案例：它是支撑 Claude 新上线的文档编辑能力 的技能之一。Claude 已经非常擅长理解 PDF，但在直接操作它们（例如填写表单）方面仍受限。这个 PDF 技能 让我们能够赋予 Claude 这些新能力。

在最简单的情况下，一个技能就是一个包含 SKILL.md file 的目录。该文件必须以 YAML 前言开头，其中包含一些必需的元数据：name 和 description。在启动时，智能体会将每个已安装技能的 name 和 description 预加载到其系统提示中。

这些元数据是渐进披露的第一层：它们为 Claude 提供恰到好处的信息，让它知道何时应该使用某个技能，而无需把全部内容加载进上下文。该文件的正文是第二层细节。如果 Claude 认为这个技能与当前任务相关，它就会通过读取完整的 SKILL.md 把技能加载进上下文。

随着技能日益复杂，它们所包含的上下文可能太多，无法放进单个 SKILL.md 中，或者只有在特定场景才相关。在这种情况下，技能可以在目录中打包额外文件，并在 SKILL.md 中按名称引用它们。这些额外的关联文件就是第三层（及更高层次）的细节，Claude 只会在需要时选择阅读。

在下图的 PDF 技能中，SKILL.md 引用了另外两个文件（reference.md 和 forms.md），技能作者选择将它们与核心的 SKILL.md 一同打包。通过把表单填写指引移动到单独的 forms.md，技能作者得以保持技能核心精简，并相信 Claude 只会在填写表单时读取 forms.md。

渐进披露是让 Agent Skills 既灵活又可扩展的核心设计原则。就像一本组织良好的手册，先是目录，再是具体章节，最后是详尽的附录，技能让 Claude 只在需要时加载信息：

拥有文件系统和代码执行工具的智能体，在处理某个任务时无需把整个技能内容都读进上下文窗口。这意味着可以被打包进技能的上下文量实际上不受限制。

操作顺序如下：

  1. 起初，上下文窗口包含核心系统提示、每个已安装技能的元数据以及用户的初始消息；
  2. Claude 通过调用 Bash 工具读取 pdf/SKILL.md，从而触发 PDF 技能；
  3. Claude 选择读取与技能一起打包的 forms.md 文件；
  4. 最后，Claude 在加载了 PDF 技能的相关指引后继续执行用户任务。

技能与代码执行

技能也可以包含代码，供 Claude 按需作为工具执行。

大型语言模型擅长许多任务，但某些操作更适合传统的代码执行。例如，通过逐 token 生成来排序列表远比直接运行排序算法昂贵。除了效率考量，许多应用场景还需要只有代码才能提供的确定性可靠性。

在我们的示例中，PDF 技能包含一个预编写的 Python 脚本，它可以读取 PDF 并提取所有表单字段。Claude 无需把脚本或 PDF 加载进上下文就能运行该脚本。而且因为代码具有确定性，这个流程始终一致且可重复。

开发与评估技能

以下是一些帮助你开始撰写和测试技能的实用指南：

  • 从评估开始： 在代表性任务上运行你的智能体，找出它在哪些地方能力不足或需要额外上下文。然后逐步构建技能来弥补这些短板。
  • 为规模而设计： 当 SKILL.md 文件变得难以驾驭时，将内容拆分成独立文件并引用它们。如果某些上下文互相独立或很少同时使用，保持路径分离能减少 token 消耗。最后，代码既可以作为可执行工具，也能充当文档。务必让 Claude 明白脚本应该直接运行还是作为参考读入上下文。
  • 站在 Claude 的角度思考： 观察 Claude 在真实场景下如何使用你的技能，并据此迭代：留意意外的使用路径或对某些上下文的过度依赖。特别关注技能的 name 和 description。Claude 会根据它们来判断是否在当前任务中触发技能。
  • 与 Claude 一起迭代： 当你和 Claude 协同处理任务时，让 Claude 把成功的方法和常见的失误提炼进技能中的可复用上下文与代码。如果它在使用技能时偏离轨道，要求它反思问题出在哪儿。这个过程能帮助你发现 Claude 实际需要的上下文，而不是事先凭空猜测。

使用 Skills 时的安全注意事项

技能通过指令和代码为 Claude 提供新能力。这让它们很强大，但也意味着恶意技能可能在使用环境中引入漏洞，或指使 Claude 外泄数据、执行意料之外的操作。

我们建议仅从可信来源安装技能。如果要安装来自不那么可信来源的技能，请在使用前彻底审计。先阅读技能中打包的文件内容，了解它的具体行为，尤其要关注代码依赖和附带的资源（如图像或脚本）。同样地，还要留意技能内的指令或代码是否要求 Claude 连接到可能不可信的外部网络资源。

Skills 的未来

Agent Skills 目前已经在 Claude.ai、Claude Code、Claude Agent SDK 和 Claude Developer Platform 上获得支持。

在接下来的数周里，我们会持续推出新功能，支持 Skills 在创建、编辑、发现、分享和使用上的全生命周期。我们尤其期待 Skills 能帮助组织和个人将自身的上下文和工作流分享给 Claude。我们还将探索 Skills 如何与 Model Context Protocol（MCP）服务器互补，教会智能体使用涉及外部工具和软件的复杂工作流。

展望未来，我们希望让智能体能够自主创建、编辑和评估 Skills，使它们能够把自身的行为模式固化成可复用的能力。

Skills 的概念简单，格式也同样简洁。这种简单性让组织、开发者和终端用户更容易构建定制化智能体，并赋予它们新的能力。

我们期待看到大家用 Skills 构建出的成果。现在就前往我们的 Skills 文档和示例集开始探索吧。
